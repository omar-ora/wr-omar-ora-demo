This tool implements the Monte Carlo Tree Search (MCTS) algorithm to determine the optimal priority sequence for assigning shared subcontractor crews across eight simultaneously active projects. When two or more projects require the same trade subcontractor at the same time, W Rondon must decide which project gets priority — a decision complicated by real-world uncertainty: a subcontractor may complete a task without interruption, or may pause mid-task to attend to a commitment with another builder before returning. This is a Risk-Aware, Multi-Project Resource-Constrained Scheduling Problem (RAPSP) — MCTS runs 5,000 directed iterations using the UCT (Upper Confidence Bound for Trees) selection criterion, simulating many possible interruption scenarios for each candidate sequence to identify the order that minimizes total expected Delay Cost. All cost fields ($) are in US dollars. Time fields in days from today (Day 0). Distance fields in miles. All outputs are computed by the algorithm — nothing is hardcoded.

Delay Cost = for each day the project is ready but the shared subcontractor has not yet arrived (up to 10 days), fixed daily overhead ($66.67 — insurance, jobsite facilities, misc., company overhead ÷ total real project duration) plus a non-linear phase-value penalty (daily phase value × [day/10]²). If the wait would exceed 10 days, W Rondon replaces the subcontractor at day 10: the Delay Cost is capped at the 10-day value, and the shared subcontractor's own schedule is unaffected (it never actually went to that site) — it remains available for the next project in the sequence. Logistics Cost = distance between sites × cost per mile — charged once per move, never per day.
Daily phase value = project square footage × cost per sq ft for that trade (see Cost per Sq Ft by Trade table) ÷ Base Duration. This is a predictive scheduling penalty, not a legal damages calculation or accounting record of actual loss.
